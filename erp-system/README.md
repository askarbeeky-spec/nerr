# Инструкция по запуску ERP системы

В данном документе описаны подробные шаги для локального запуска проекта. Проект состоит из backend-части (Node.js, база данных в Docker) и frontend-части (React Native / Expo).

## 📋 Предварительные требования

Убедитесь, что у вас установлены следующие программы:
1. **Node.js** (рекомендуется версия 18 или выше)
2. **Docker Desktop** (для запуска базы данных и/или бэкенда)
3. **Expo Go** — приложение на вашем мобильном телефоне (доступно в App Store и Google Play)
4. **Git** (опционально, для управления версиями)

---

## 🛠️ Шаг 1: Запуск Backend-части (и базы данных) через Docker

Бэкенд и база данных (PostgreSQL) могут быть запущены в контейнерах Docker для удобства.

1. Откройте терминал и перейдите в папку бэкенда:
   ```bash
   cd erp-system/backend
   ```

2. Убедитесь, что приложение **Docker Desktop** запущено на вашем компьютере.

3. Создайте файл `.env` в папке `backend` (если его нет) и добавьте необходимые переменные окружения (например, данные для подключения к БД, секретные ключи для JWT и т.д.).

4. Запустите контейнеры с помощью Docker Compose:

   ```bash
   cd (полный путь к проекту)
   docker-compose up -d
   ```
   пример (cd C:\Users\имя пользователя\Desktop\Нуртай\erp-system\frontend
$env:EXPO_PUBLIC_API_URL="http://ваш апи :3000/api"; npx expo start --port 8082)
   *(Флаг `-d` запустит контейнеры в фоновом режиме. Если вы хотите видеть логи бэкенда в реальном времени, запустите без `-d`: `docker-compose up`)*

5. Чтобы остановить контейнеры в будущем, используйте команду:
   ```bash
   docker-compose down
   ```

> **Примечание:** Если вы запускаете базу данных через Docker, а бэкенд локально через Node.js, выполните `npm install` и `npm run dev` в папке `backend` после поднятия базы данных.

---

## 📱 Шаг 2: Запуск Frontend-части (Приложение Expo)

Фронтенд проекта написан на React Native с использованием фреймворка Expo.

1. Откройте **новый** терминал (оставив терминал с бэкендом работать) и перейдите в папку фронтенда:
   ```bash
   cd erp-system/frontend
   ```

2. Установите все необходимые зависимости (нужно сделать только один раз после клонирования проекта):
   ```bash
   npm install
   ```

3. Настройте подключение к бэкенду. Создайте файл `.env` в папке `frontend` и укажите IP-адрес вашего компьютера (важно для работы на реальном устройстве), например:
   ```env
   EXPO_PUBLIC_API_URL=http://192.168.x.x:3000/api
   ```
   *(Замените `192.168.x.x` на ваш локальный IPv4 адрес)*

4. Запустите сервер разработки Expo:
   ```bash
   npx expo start
   ```
   *(Или `npm start`, если такой скрипт прописан в package.json)*

## 🚀 Шаг 3: Как открыть приложение в Expo Go

После успешного выполнения команды `npx expo start`, в вашем терминале (и/или в открывшейся вкладке браузера) появится большой **QR-код**.

### Для пользователей iPhone (iOS):
1. Убедитесь, что ваш iPhone и компьютер подключены к **одной Wi-Fi сети**.
2. Откройте стандартное приложение **Камера** на телефоне.
3. Наведите камеру на QR-код в терминале.
4. Появится уведомление с предложением открыть проект в приложении **Expo Go**. Нажмите на него.

### Для пользователей Android:
1. Убедитесь, что ваш Android-смартфон и компьютер подключены к **одной Wi-Fi сети**.
2. Откройте приложение **Expo Go**.
3. На главном экране приложения нажмите кнопку **Scan QR Code**.
4. Наведите камеру на QR-код в терминале.

> **Возможные проблемы:** 
> Если Expo Go выдает ошибку сети (Network Error), убедитесь, что:
> - Компьютер и телефон находятся в одной Wi-Fi сети.
> - Антивирус или брандмауэр (Windows Defender) не блокирует порты Node.js (обычно порт 8081).
> - Попробуйте перезапустить Expo с флагом для локальной сети: `npx expo start --lan` или для туннеля: `npx expo start --tunnel`.

## 🛑 Остановка проекта

- Чтобы остановить фронтенд, вернитесь в терминал с Expo и нажмите `Ctrl + C`.
- Чтобы остановить бэкенд в Docker, выполните `docker-compose down` в папке `backend`.
